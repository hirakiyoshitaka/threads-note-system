# CONTENT RULES

## 基本方針

よんぴーの投稿は、派手な成功談ではなく、実務でAIを使いながら前に進む人の記録として書きます。読者が「この人なら自分のつまずきも分かってくれそう」と感じる温度にします。

## 口調

- 丁寧だが、かしこまりすぎない。
- 上から教えるより、横で一緒に進む。
- 断定よりも、実体験からの気づきとして書く。
- 難しい言葉より、仕事や日常の言葉を使う。

## 文体

- 1文を短めにする。
- 冒頭で読者の悩みを言い切る。
- 自分の失敗や勘違いを1つ入れる。
- 最後はnoteへの自然な案内で終える。
- 改行を使い、スマホで読みやすくする。

## 避ける表現

- 絶対に稼げる。
- 誰でも簡単。
- 今だけ、限定、急いで。
- AIを使わない人は遅れている。
- 情弱、勝ち組、負け組。
- 実績の捏造や、根拠のない数字。
- 読者を不安で追い込む表現。

## 読者に刺さる型

### 1. 共感フック

読者が心の中で言っていることを最初に置きます。

例:

`AIを使いたいのに、画面の前で手が止まることがあります。`

### 2. 自分の失敗談

発信者が先に失敗を出すと、読者は安心して読み進められます。

例:

`私も最初は、便利そうな機能を全部試して、結局どれも続きませんでした。`

### 3. 今日の学び

失敗から得た小さな学びを1つに絞ります。

例:

`大事なのは機能の数ではなく、今日の仕事に1つだけ使うことでした。`

### 4. 読者への置き換え

読者が自分の仕事や生活に置き換えられる一文を入れます。

例:

`メール、資料、投稿文。どれか1つに絞るだけで始めやすくなります。`

### 5. noteへの自然なCTA

Threadsで完結させすぎず、noteで詳しい手順が読めることを伝えます。

例:

`私が最初に使った手順はnoteに整理しました。必要な方だけどうぞ。`
