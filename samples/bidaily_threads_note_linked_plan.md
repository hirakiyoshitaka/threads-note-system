# 2日おき Threads -> note 連動サンプル

## 前提

- 既存レーン: 毎日note自動配信は今まで通り継続。
- 新レーン: 2日おきにThreadsで悩み予告を出し、対応するnote下書きを生成する。
- 公開前に必ず本人が確認・リライトする。

## Cycle 1

### Threads悩み予告

AIを使いたいのに、最初の一言で止まることがあります。

次回、こんな悩みありませんか？
「AIに何を頼めばいいのか分からない」

私も最初は、便利そうな画面を開いて閉じるだけでした。実務で少し進んだきっかけは、AIに頼む前に仕事を1つに絞ったことです。

次のnoteで、私が最初に使った頼み方を整理します。

### note下書きテーマ

- タイトル案: `AIに何を頼めばいいか分からない人へ。最初の1つの決め方`
- 答える悩み: AIの使い道が分からず、画面の前で止まる。
- 本文の核: 今日の仕事を1つ選び、AIに渡す前に目的を1行で書く。
- 予約判断: 本人確認後、既存note予約フローへ渡す。

## Cycle 2

### Threads悩み予告

AIに文章を頼んだのに、直す時間の方が長くなることがあります。

次回、こんな悩みありませんか？
「AIの文章が自分の言葉にならない」

私も最初は、出てきた文章をそのまま使おうとして違和感が残りました。実務では、先に読者と口調を決めた方が修正が少なくなりました。

次のnoteで、私が使っている修正前提の頼み方を書きます。

### note下書きテーマ

- タイトル案: `AIの文章が自分の言葉にならない時に、先に決める3つのこと`
- 答える悩み: AI文章が便利だが、自分らしくなくて投稿しづらい。
- 本文の核: 読者、口調、避ける表現を先に決める。
- 予約判断: 本人確認後、既存note予約フローへ渡す。

## Cycle 3

### Threads悩み予告

AIを使い始めても、続かないことがあります。

次回、こんな悩みありませんか？
「毎日使おうと思っても、気づくと元のやり方に戻っている」

私もそうでした。大きく変えようとすると疲れます。続いたのは、毎日くり返す作業を1つだけAIに手伝わせるやり方でした。

次のnoteで、続けるための小さな仕組みをまとめます。

### note下書きテーマ

- タイトル案: `AIが続かない人へ。毎日1つだけ使う仕組みの作り方`
- 答える悩み: AIを試しても習慣にならない。
- 本文の核: 毎日必ず発生する作業を1つ選び、固定の頼み方を作る。
- 予約判断: 本人確認後、既存note予約フローへ渡す。
