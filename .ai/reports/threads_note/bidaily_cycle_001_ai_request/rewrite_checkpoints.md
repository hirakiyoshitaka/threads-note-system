# リライト確認ポイント

## 事実確認

- [ ] 「57歳からAIを実務で使っている」という立ち位置が本人の実感と合っている。
- [ ] 「画面を開いて閉じるだけだった」という表現が、自分の体験として言える。
- [ ] メール、会議メモ、投稿文、note見出しの例が実際の用途とズレていない。
- [ ] 実績、数字、成果を盛っていない。

## 既存レーンとの衝突確認

- [ ] 既存の毎日note自動配信テーマと同じ内容を重複公開しない。
- [ ] 既存の予約済みnote記事を上書きしない。
- [ ] 既存レーンの配信スケジュールを変更しない。
- [ ] この下書きは確認後にだけ予約投稿フローへ渡す。

## 文体確認

- [ ] 上から教える口調になっていない。
- [ ] 同世代やAI初心者を不安で煽っていない。
- [ ] 「誰でも簡単」「絶対できる」「稼げる」などの断定表現がない。
- [ ] 1文が長すぎる箇所を短くする。
- [ ] 自分の言葉に置き換える。

## Threadsとの連動確認

- [ ] Threadsの問い「AIに何を頼めばいいか分からない」にnote本文が答えている。
- [ ] note冒頭でThreadsの悩みを再掲している。
- [ ] Threadsで約束した「最初に使った頼み方」が本文にある。
- [ ] note末尾に次回Threads予告につながる問いがある。

## 追加すると良い本人メモ

- 実際に最初にAIへ頼んだ仕事。
- うまくいかなかった依頼文の具体例。
- 今も使っている依頼文の定番。
- note読者に渡したいチェックリスト。
